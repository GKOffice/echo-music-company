# WWW Redirect Fix — Manual Steps
**Problem:** `www.melodio.io` hits Cloudflare but Railway returns 404 because Railway Free plan
only allows 1 custom domain per service. The fix is a Cloudflare Redirect Rule so all `www`
traffic 301-redirects to `melodio.io`.

---

## What You Need
- Cloudflare account login (or API token with **Zone.Page Rules** or **Zone.Cache Rules** write permission)
- Zone ID for `melodio.io` (found in Cloudflare dashboard → select melodio.io → Overview → right sidebar)

---

## Option 1 — Cloudflare Dashboard (Easiest, ~2 minutes)

1. Log in → **cloudflare.com/dashboard**
2. Select the **melodio.io** zone
3. Left sidebar → **Rules → Redirect Rules**
4. Click **Create rule**
5. Fill in:
   - **Rule name:** `www to apex redirect`
   - **If… Field:** `Hostname` · **Operator:** `equals` · **Value:** `www.melodio.io`
   - **Then…** → `Dynamic redirect`
   - **Expression:** `concat("https://melodio.io", http.request.uri.path)`
   - **Status code:** `301`
6. Click **Deploy**

That's it. No Railway changes needed.

## Option 2 — Cloudflare API (for automation / scripting)

### Step 1 — Get your Zone ID
```bash
curl -s https://api.cloudflare.com/client/v4/zones \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json" | jq '.result[] | {name: .name, id: .id}'
```

### Step 2 — Create the redirect rule
```bash
ZONE_ID="<your_zone_id>"
CF_TOKEN="<your_api_token>"

curl -s -X POST \
  "https://api.cloudflare.com/client/v4/zones/${ZONE_ID}/rulesets/phases/http_request_dynamic_redirect/entrypoint/rules" \
  -H "Authorization: Bearer ${CF_TOKEN}" \
  -H "Content-Type: application/json" \
  --data '{
    "description": "Redirect www to apex",
    "expression": "(http.host eq \"www.melodio.io\")",
    "action": "redirect",
    "action_parameters": {
      "from_value": {
        "status_code": 301,
        "target_url": {
          "expression": "concat(\"https://melodio.io\", http.request.uri.path)"
        },
        "preserve_query_string": true
      }
    },
    "enabled": true
  }'
```

### Step 3 — Verify (after ~30 seconds)
```bash
curl -sI https://www.melodio.io | grep -i location
# Should return: location: https://melodio.io/
```

## Option 3 — Page Rule (Legacy, still works)

1. Cloudflare Dashboard → melodio.io → **Rules → Page Rules**
2. **Create Page Rule**
3. URL: `www.melodio.io/*`
4. Setting: **Forwarding URL** → 301
5. Destination: `https://melodio.io/$1`
6. Save and Deploy

> Note: Cloudflare Free plan includes 3 Page Rules — this uses one.

---

## Recommended
**Use Option 1 (Dashboard → Redirect Rules)** — it's free, unlimited, and faster than Page Rules.
